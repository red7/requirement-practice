# 深度场景化 Prompt 系统说明

## 🎯 系统特点

### 1. **真实业务场景**
每个行业都有完整的背景故事：
- 具体的角色定位（产品经理、信息科主任、运营总监...）
- 真实的业务问题（转化率下降、挂号难、购物车放弃...）
- 明确的成功指标（提升转化率10%、缩短审批时间...）

### 2. **渐进式信息释放机制**

#### 小白级别（1星难度）
- 会主动说出所有关键信息
- 表达清晰、逻辑完整
- 适合快速熟悉场景

#### 职场现实（3星难度）
- **第1-3轮**：只说表面痛点 + 30%噪音（抱怨、闲聊）
- **第4-6轮**：用户问到"数据"、"指标"等关键词，才透露核心数据
- **第7轮+**：用户问到"为什么"、"根本原因"，才说深层问题
- **隐藏约束**：只有问到"合规"、"系统"、"团队"等，才会提及

### 3. **三类隐藏约束**

每个场景都包含3个隐藏约束，需要用户主动挖掘：

| 类型 | 触发词 | 示例 |
|------|--------|------|
| 监管/合规约束 | 合规、监管、法规、政策 | "必须双录，AI推荐需要可解释性" |
| 技术约束 | 系统、技术、数据、接口 | "数据分散在3个老系统，核心是COBOL代码" |
| 组织约束 | 团队、人力、预算、时间 | "只给3个月，只有2个开发，预算砍30%" |

### 4. **真实数据和指标**

不再是模糊的"效率低"，而是：
- 金融：转化率从12% → 4.2%，停留时长从8分钟 → 2分钟
- 医疗：82%用户尝试3次以上，每天200+投诉
- 电商：加购转化率45%，购物车平均8.3件只买1.2件

## 🎮 5个完整场景

### 金融 - 理财产品推荐系统
**角色**：某互联网银行产品经理
**痛点层级**：
1. 表面：推荐不精准
2. 核心：转化率从12%跌至4.2%
3. 深层：算法只看年龄收入，不考虑风险偏好

**隐藏约束**：
- 监管：必须双录，AI需要可解释性
- 技术：数据分散3个老系统，核心是COBOL代码
- 组织：3个月上线，只有2个开发

### 医疗 - 门诊挂号系统
**角色**：某三甲医院信息科主任
**痛点层级**：
1. 表面：专家号秒光
2. 核心：82%用户需尝试3次以上，每天200+投诉
3. 深层：先到先得无分诊，黄牛脚本抢号

**隐藏约束**：
- 监管：卫健委要求实名制，数据不能出内网
- 技术：HIS系统10年前的，供应商倒闭了
- 组织：医生不配合，担心AI误诊被追责

### 电商 - 购物车转化优化
**角色**：某电商平台运营总监
**痛点层级**：
1. 表面：购物车放弃率70%
2. 核心：加购→下单45%，下单→支付60%
3. 深层：运费不透明、优惠券复杂、支付要跳3页

**隐藏约束**：
- 商业：改运费规则商家会炸锅，上次改佣金200商家威胁撤店
- 技术：支付系统外包，接口响应3-5秒
- 组织：运营、技术、商家运营KPI完全不一样

### 合规 - 审批流程优化
**角色**：某金融机构合规部经理
**痛点层级**：
1. 表面：审批慢，业务部门天天催
2. 核心：5道关卡，平均5天，积压80+项目
3. 深层：反洗钱靠Excel手工操作，审批标准不统一

**隐藏约束**：
- 监管：必须留痕可追溯，AI责任不明确
- 技术：数据敏感权限严，外部数据库无API
- 组织：只有6个专家，AI培训无人做，老员工抵触

### 运动健康 - 数据追踪系统
**角色**：某运动健康App产品负责人
**痛点层级**：
1. 表面：用户换App，数据割裂
2. 核心：30天留存18%，用户看不到长期进步
3. 深层：缺个性化指导，只记运动不看饮食睡眠

**隐藏约束**：
- 数据：Apple/小米/华为API不统一，用户不愿授权
- 专业：运动处方是医疗行为，受伤有法律责任
- 商业：付费率不到5%，获客成本80元客单价200元

## 💡 训练价值提升

### 之前（简单版）
```
AI: "理财产品推荐不够精准"
用户: "为什么？"
AI: "算法不好"
```
❌ 太简单，学不到东西

### 现在（深度版）
```
AI: "唉，别提了...最近用户投诉特别多，说我们推荐的产品不靠谱...
    对了，下午还有个会要开，咱们抓紧说..."

用户: "能具体说说有多严重吗？有数据吗？"

AI: "数据啊...让我看看...转化率从去年的12%掉到现在4.2%了，
    用户平均停留时长也从8分钟降到2分钟，68%的人看完就退出了..."

用户: "为什么会这样？根本原因是什么？"

AI: "说实话，咱们的推荐算法太简单了，就看用户年龄和收入，
    完全没考虑风险偏好啊、投资经验啊这些。而且风险评估问卷
    还是5年前设计的，早就过时了..."

用户: "有什么技术限制吗？"

AI: "哦对了，这个问题大了！用户数据散落在3个不同系统里——
    CRM、交易系统、风控系统，根本没有统一的用户画像。
    而且核心交易系统还是15年前的COBOL代码，改接口风险特别大..."

用户: "有合规方面的约束吗？"

AI: "你算问到点子上了！银保监会最新规定，推荐理财产品必须双录，
    而且推荐理由要可追溯可解释。如果用AI黑盒模型，
    可能根本过不了合规审查..."
```
✅ 真实、深入，训练价值高

## 📊 需求分析能力训练

使用这个系统，可以训练：

### 1. **提问技巧**
- ❌ "有什么问题？" → 太泛
- ✅ "能具体说说数据指标吗？转化率是多少？"

### 2. **信息挖掘**
- 学会追问"为什么"、"具体流程"、"有什么限制"
- 识别表面需求 vs 深层需求

### 3. **约束识别**
- 主动询问："有合规要求吗？"
- 主动询问："现有系统架构是怎样的？"
- 主动询问："团队资源和时间限制是什么？"

### 4. **结构化思维**
- 痛点 → 数据 → 原因 → 约束 → 方案
- 业务目标 → 技术可行性 → 组织可行性
